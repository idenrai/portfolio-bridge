# i18n System

## Supported Locales

| Code | Language | Flag | `LANG_NAMES` value |
| --- | --- | --- | --- |
| `ko` | Korean / 한국어 | 🇰🇷 | `"한국어"` |
| `en` | English | 🇺🇸 | `"English"` |
| `ja` | Japanese / 일본어 | 🇯🇵 | `"日本語"` |
| `de` | German / 독일어 | 🇩🇪 | `"Deutsch"` |

Active locale is stored in `useLanguageStore` and persisted to localStorage.

현재 언어는 `useLanguageStore`에 저장되며 localStorage에 유지됩니다.

## File Structure

파일 구조는 다음과 같습니다.

```text
src/i18n/
├── types.ts     # Translations interface — all keys typed here / 모든 키 타입 정의
├── ko.ts        # Korean translations / 한국어 번역
├── en.ts        # English translations / 영어 번역
├── ja.ts        # Japanese translations / 일본어 번역
├── de.ts        # German translations / 독일어 번역
└── index.ts     # TRANSLATIONS map, LANG_NAMES, LANG_LOCALES exports
```

## Usage

컴포넌트에서의 사용 방법입니다.

```typescript
const t = useT(); // hook from src/hooks/useT.ts

t.some_key        // string
t.fn_key(n)       // parameterized: (n: number) => string
```

`useT()` reads the active locale from `useLanguageStore` and returns the matching translation object.

`useT()`는 `useLanguageStore`에서 현재 언어를 읽어 해당 번역 객체를 반환합니다.

## Key Naming Conventions

Keys use `snake_case`. Prefixes group related keys.

키는 `snake_case`를 사용하며, 접두사로 관련 키를 그룹화합니다.

| Prefix / 접두사 | Area / 영역 |
| --- | --- |
| `dashboard_*` | Dashboard page / 대시보드 |
| `assets_*` | Asset management / 자산 관리 |
| `gurus_*` | Gurus page / 투자 구루 |
| `settings_*` | Settings page / 설정 |
| `profile_*` | User profile / 사용자 프로필 |
| `at_col_*` | Asset table column headers / 자산 테이블 컬럼 헤더 |
| `guru_name_*` | Guru display names / 구루 표시명 |
| `guru_philosophy_*` | Guru philosophy text / 구루 철학 텍스트 |
| `guru_quotes_*` | Guru quotes / 구루 명언 |
| `about_*` | About page / 소개 페이지 |

Parameterized translations are typed as functions: `(n: number) => string`.

파라미터가 있는 번역은 함수 타입으로 정의됩니다: `(n: number) => string`.

## Adding a New Key

새 키를 추가하는 절차입니다.

1. Add the key and type to `src/i18n/types.ts` → `Translations` interface. / `Translations` 인터페이스에 키와 타입 추가.
2. Add the key with value to **all 4** locale files: `ko.ts`, `en.ts`, `ja.ts`, `de.ts`. / **4개** 언어 파일 모두에 값 추가.
3. Use in components via `t.new_key`. / 컴포넌트에서 `t.new_key`로 사용.

> TypeScript will produce a compile error if a key is present in `types.ts` but missing from any locale file.
>
> `types.ts`에 키가 있지만 언어 파일에 누락된 경우 TypeScript 컴파일 오류가 발생합니다.

## Guru Philosophy & Quotes Format

각 구루의 철학은 `guru_philosophy_<id>`, 명언은 `guru_quotes_<id>` 키로 명확히 분리하여 관리합니다.

Rules:

- `guru_philosophy_<id>`: 철학 원칙 5~6개.
  - Use `•` as the bullet character. / 불릿 문자로 `•` 사용.
  - Use `\n` for line breaks (matching the concatenation style in existing locale files). / 줄바꿈에 `\n` 사용.
- `guru_quotes_<id>`: 명언 문장. 여러 개일 경우 `\n`으로 구분.
  - 명언 문자열 자체에는 앞뒤에 인용 부호(`"` 또는 `'`)를 직접 포함하지 않습니다. UI 컴포넌트가 렌더링 시 디자인 요소로 장식용 따옴표를 추가합니다.
- Keep investment terminology consistent across locales (e.g., ROIC, Margin of Safety, Risk Parity). / 투자 도메인 용어 일관성 유지.
- Prefer natural, domain-accurate phrasing over machine-literal translation. / 직역보다 투자 문맥에 자연스러운 의역 우선.

## Fallback Behavior

If a locale-specific key is missing at runtime, the `GurusPage` component falls back
to the English value for guru philosophy keys.

런타임에 언어별 키가 누락된 경우, `GurusPage` 컴포넌트는 구루 철학 키에 대해 영어 값으로 폴백합니다.

## AI-Facing Text

Guru philosophy text injected into AI prompts is always the **English version** (`philosophyEn`),
regardless of the active UI language. This ensures cross-language consistency in prompt quality.

AI 프롬프트에 주입되는 구루 철학 텍스트는 현재 UI 언어와 무관하게 항상 **영어 버전** (`philosophyEn`)을 사용합니다.
이는 언어 간 프롬프트 품질 일관성을 보장합니다.
