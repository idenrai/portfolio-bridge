# i18n System

## Supported Locales

| Code | Language | Flag | `LANG_NAMES` value |
| --- | --- | --- | --- |
| `ko` | Korean | 🇰🇷 | `"한국어"` |
| `en` | English | 🇺🇸 | `"English"` |
| `ja` | Japanese | 🇯🇵 | `"日本語"` |
| `de` | German | 🇩🇪 | `"Deutsch"` |

Active locale is stored in `useLanguageStore` and persisted to localStorage.

## File Structure

```text
src/i18n/
├── types.ts     # Translations interface — all keys typed here
├── ko.ts        # Korean translations
├── en.ts        # English translations
├── ja.ts        # Japanese translations
├── de.ts        # German translations
└── index.ts     # TRANSLATIONS map, LANG_NAMES, LANG_LOCALES exports
```

## Usage

```typescript
const t = useT(); // hook from src/hooks/useT.ts

t.some_key        // string
t.fn_key(n)       // parameterized: (n: number) => string
```

`useT()` reads the active locale from `useLanguageStore` and returns the matching translation object.

## Key Naming Conventions

Keys use `snake_case`. Prefixes group related keys:

| Prefix | Area |
| --- | --- |
| `dashboard_*` | Dashboard page |
| `assets_*` | Asset management page |
| `gurus_*` | Gurus page |
| `settings_*` | Settings page |
| `profile_*` | User profile section |
| `at_col_*` | Asset table column headers |
| `guru_name_*` | Guru display names |
| `guru_philosophy_*` | Guru philosophy text |
| `about_*` | About page |

Parameterized translations are typed as functions: `(n: number) => string`.

## Adding a New Key

1. Add the key and type to `src/i18n/types.ts` → `Translations` interface.
2. Add the key with value to **all 4** locale files: `ko.ts`, `en.ts`, `ja.ts`, `de.ts`.
3. Use in components via `t.new_key`.

> TypeScript will produce a compile error if a key is present in `types.ts` but missing from any locale file.

## Guru Philosophy Format

Each `guru_philosophy_<id>` key must follow a strict **6-line structure**:

```text
• [Principle 1]\n
• [Principle 2]\n
• [Principle 3]\n
• [Principle 4]\n
• [Principle 5]\n
• [Quote label]: "[quote text]"
```

Quote label by locale:

| Locale | Label |
| --- | --- |
| `en` | `Quote:` |
| `ko` | `명언:` |
| `ja` | `名言：` |
| `de` | `Zitat:` |

Rules:

- Use `•` as the bullet character.
- Use `\n` for line breaks (matching the concatenation style in existing locale files).
- Keep investment terminology consistent across locales (e.g., ROIC, Margin of Safety, Risk Parity).
- Prefer natural, domain-accurate phrasing over machine-literal translation.

## Fallback Behavior

If a locale-specific key is missing at runtime, the `GurusPage` component falls back
to the English value for guru philosophy keys.

## AI-Facing Text

Guru philosophy text injected into AI prompts is always the **English version** (`philosophyEn`),
regardless of the active UI language. This ensures cross-language consistency in prompt quality.
