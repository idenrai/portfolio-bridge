# 오류 해결 워크플로우 (Troubleshoot)

**Activation:** 이 워크플로우는 사용자가 `/troubleshoot` 명령어를 사용하거나 깊이 있는 오류 원인 분석 및 해결을 요청했을 때 활성화됩니다.

오류 원인 분석 및 해결 요청을 받으면 **반드시 다음 4단계**를 순서대로 거쳐 문제를 해결해야 합니다.

## 1. 【조사】 (Investigation)
- 오류의 정확한 증상과 원인을 파악하기 위해 에이전트에 내장된 **웹 검색 도구(`search_web`)**를 사용해 조사합니다.
- 단순한 코드 구문 오류가 아닌 외부 라이브러리, 배포 환경(Vercel, Tauri 등), API 변경점 관련 이슈라면 반드시 검색을 통해 사례를 확보하세요.

## 2. 【지식】 (Knowledge)
- **Context7 MCP 도구**를 호출하여 최신 공식 문서(Docs), 깃허브 이슈 레퍼런스, 라이브러리 사용법을 참조합니다.
- 검색으로 얻은 단편적인 정보에만 의존하지 말고, 신뢰할 수 있는 최신 문서를 통해 '올바른 작성법'이 무엇인지 교차 검증(Cross-check)해야 합니다.

## 3. 【논리】 (Logic)
- 본격적인 수정에 앞서 **Sequential Thinking MCP 도구**를 활성화합니다.
- "왜 이 에러가 발생했는가?", "어떤 부작용(Side Effect)이 있을 수 있는가?", "어떤 방식으로 수정하는 것이 가장 이상적인가?"에 대해 여러 단계에 걸쳐 체계적이고 깊이 있게 사고(Thinking) 과정을 거치세요.

## 4. 【실행】 (Execution)
- 위 3단계를 통해 원인 규명과 논리적 해결 방안이 완벽하게 도출된 이후에만 코드를 수정합니다.
- 내장된 **파일 제어 도구(`replace_file_content`, `run_command` 등)**를 사용해 소스 코드와 설정 파일을 자동으로, 그리고 정확하게 수정합니다.

> **주의사항**: 위 4단계의 과정을 진행할 때, 각각 어떤 단계(조사, 지식, 논리, 실행)를 밟고 있는지 사용자에게 명확히 요약하여 알려주어야 합니다.
